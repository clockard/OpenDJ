The ldifsearch Utility

The ldifsearch utility may be used to perform searches in an LDIF file. It is very similar to the ldapsearch tool, with the exception that it does not perform any LDAP communication but rather operates on a local LDIF file.

The command-line usage for this tool is as follows:
ldifsearch {options} [filter] [attributes...]

The complete set of named arguments that may be used with the ldifsearch tool is as follows:
  • -l {path}
    --ldifFile {path}

    This argument specifies the path to the LDIF file containing the data to be searched. Multiple LDIF files may be specified by providing this option multiple times. This argument is required.

  • -b {dn}
    --baseDN {dn}

    This argument specifies the base DN to use for the search operation. Multiple base DNs may be provided by using this option multiple times, and if multiple values are provided then an entry will be examined if it is within the scope of any of the search bases. If no search base is provided, then any entry contained in the LDIF file(s) will be considered in the scope of the search.

  • -s {scope}
    --searchScope {scope}

    This argument specifies the scope to use for the search operation. Its value must be one of "base", "one", "sub", or "subordinate". If this argument is not provided, then a default value of "sub" will be used.

  • -f {path}
    --filterFile {path}

    This argument specifies the path to a file containing one or more filters to use when processing the search operation. If there are to be multiple filters, then the file should be structured with one filter per line. If this option is used, then any trailing arguments will be treated as separate attributes; otherwise the first trailing argument must be the search filter.

  • -o {path}
    --outputFile {path}

    This argument specifies the path to the output file that should be written containing the entries matching the provided search criteria. If this argument is not provided, then matching entries will be written to standard output.

  • -O
    --overwriteExisting

    This argument indicates that if the specified output file already exists that it should be overwritten rather than appended to. This is only applicable if the --outputFile argument is used.

  • -z {numEntries}
    --sizeLimit {numEntries}

    This argument specifies the maximum number of matching entries that should be allowed. Once this number of matching entries have been identified, then the search will end.

  • -t {numSeconds}
    --timeLimit {numSeconds}

    This argument specifies the maximum length of time in seconds that should be spent performing the searches. Once this length of time has elapsed, then the search will end.

  • -T
    --dontWrap

    This argument indicates that long lines should not be wrapped when displaying matching entries. If this is not provided, then long lines will be wrapped (in a manner compatible with the LDIF specification) to fit on an 80-column terminal.

  • -H
    --help

    This argument will display usage information for the ldifsearch utility and will exit without attempting to perform any additional processing.
