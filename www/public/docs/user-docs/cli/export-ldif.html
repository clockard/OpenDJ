The export-ldif Utility

The export-ldif utility may be used to export the contents of a Directory Server backend in LDIF format.

In order to use this tool, both the --backendID and the --ldifFile arguments must be provided. The complete set of arguments that may be used is as follows:
  • -n {backendID}
    --backendID {backendID}

    This argument specifies the backend ID of the backend containing the data to be exported. The available backends in the server may be determined using the list-backends tool. This is a required argument.

  • -l {path}
    --ldifFile {path}

    This argument specifies the path to the LDIF file to which the export data should be written. This is a required argument.

  • -a
    --appendToLDIF

    This argument indicates that the server should append to an existing LDIF file rather than overwriting it. If this argument is not provided, then the server will overwrite the specified LDIF file if it already exists.

  • -b {baseDN}
    --includeBranch {baseDN}

    This argument specifies the base DN for a branch of the data that should be included in the export. This argument may be used multiple times to specify multiple exclude base DNs. If this argument is provided, then entries contained in the backend that are not at or below one of the provided include base DNs will be skipped.

  • -B {baseDN}
    --excludeBranch {baseDN}

    This argument specifies the base DN for a branch of the data that should be excluded from the export. This argument may be used multiple times to specify multiple exclude base DNs. If this argument is provided, then any entries contained in the backend that are at or below one of the provided exclude base DNs will be skipped.

    Note that the use of the --excludeBranch option takes precedence over the --includeBranch option, so if an entry is at or below a DN contained in both the include and exclude lists, it will not be included. This capability makes it possible to include data for only part of a branch (e.g., all entries below "dc=example,dc=com" except those below "ou=People,dc=example,dc=com").
  • -i {attribute}
    --includeAttribute {attribute}

    This argument specifies the name of an attribute that should be included in the export. This argument may be used multiple times to specify multiple include attributes. If this argument is provided, then any attributes not listed in this set will be omitted from the entries that are exported.

  • -e {attribute}
    --excludeAttribute {attribute}

    This argument specifies the name of an attribute that should be excluded from the export. This argument may be used multiple times to specify multiple exclude attributes. If this argument is provided, then any attributes listed in this set will be omitted from the entries that are exported.

  • -I {filter}
    --includeFilter {filter}

    This argument specifies a search filter that may be used to identify entries that should be included in the export. This argument may be used multiple times to specify multiple include filters. If this argument is provided, then any entry in the backend that does not match an include filter will be skipped.

  • -E {filter}
    --excludeFilter {filter}

    This argument specifies a search filter that may be used to identify entries that should be excluded from the export. This argument may be used multiple times to specify multiple exclude filters. If this argument is provided, then any entry in the backend that matches an exclude filter will be skipped.

    Note that the use of the --excludeFilter argument takes precedence over the --includeFilter option, so if an entry matches filters in the include and exclude lists, then the entry will be skipped.

  • -w {column}
    --wrapColumn {column}

    This argument specifies the column at which the server should wrap long lines when writing to the LDIF file. A value of zero indicates that the data should not be wrapped.

  • -c
    --compress

    This argument indicates that the server should compress the LDIF data as it is written. The data will be compressed using the GZIP format, which is the format used by the --isCompressed option of the import-ldif tool.

  • -y
    --encryptLDIF

    This argument indicates that the server should encrypt the LDIF data as it is written. The data will be encrypted in a format that may be read by the --isEncrypted argument of the import-ldif tool.

  • -s
    --signHash

    This argument indicates that the server should generate a signed hash based on the contents of the exported LDIF data. This may be used when importing the data to ensure that the content has not been altered since it was exported.

  • -H
    --help

    This argument will display usage information for the export-ldif utility and will exit without attempting to export any data.
