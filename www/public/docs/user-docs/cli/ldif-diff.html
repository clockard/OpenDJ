The ldif-diff Utility

The ldif-diff utility may be used to identify the differences between two LDIF files and output the result in LDIF change form. The resulting output may be applied to the source LDIF to result in the contents of the target LDIF file. Also note that the change information will be written so that it will contain all information necessary for someone to reverse the changes if necessary. For modify operations, this means that the replace operation will not be used, but rather only sets of value add and delete changes. Further, for delete operations, the contents of the entry that has been removed will be included in the changes output in the form of comments.

Note that this utility was only designed to work on small data sets, and therefore is only suitable for use in cases in which both the source and target data sets may fit entirely in memory at the same time. It is not intended for use on large data sets that cannot fit in available memory. The complete set of arguments that may be used with the ldif-diff tool is:
  • -s {path}
    --sourceLDIF {path}

    This argument specifies the path to the source LDIF file, which contains the original data with no changes applied. This argument is required.

  • -t {path}
    --targetLDIF {path}

    This argument specifies the path to the target LDIF file, for which the differences from the source LDIF should be identified. This argument is required.

  • -o {path}
    --outputLDIF {path}

    This argument specifies the path to the file that should be written with information about the changes between the source and target LDIF data. If this is not provided, then the change information will be written to standard output.

  • -O
    --overwriteExisting

    This argument indicates that if the specified output file already exists that it should be overwritten rather than appending to it. It is only applicable if the --outputLDIF option is used.

  • -H
    --help

    This argument will display usage information for the ldif-diff utility and will exit without attempting to perform any additional processing.
