The restore Utility

The restore utility may be used to restore a previous backup taken of a Directory Server backend. Only one backend may be restored at a time with this command.

In order to use this tool, the --backupDirectory option must be used to specify the location of the backup files. The complete set of arguments that may be used includes:
  • -d {path}
    --backupDirectory {path}

    This argument specifies the path to the directory that contains the backup archive. This directory must exist and must contain a backup descriptor file and one or more backups for a given backend. The backup descriptor file will be read to obtain information about the available backups and the options used to create them. This is a required argument.

  • -l
    --listBackups

    This argument indicates that the server should display information about the available backups contained in the backup directory. It will cause the tool to exit without performing any restore.

  • -I {backupID}
    --backupID {backupID}

    This argument may be used to specify the backup ID of the backup to be restored. If this option is not provided, then the latest backup contained in the backup directory will be restored.

  • -n
    --dry-run

    This argument may be used to specify that the server should verify that the specified backup is valid (i.e., ensure that it appears to be a valid archive, and that any hash and/or signature matches its contents). It will not actually attempt to restore the backup.

  • -H
    --help

    This argument will display usage information for the restore utility and will exit without attempting to perform a restore.
