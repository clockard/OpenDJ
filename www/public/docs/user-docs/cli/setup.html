The setup Utility

The setup utility may be used to configure a Directory Server instance for use, including specifying the port on which it will listen, the DN and password for the initial root user, the base DN for the directory data, and the manner in which the database should be populated. It may be run in one of three modes:
  • As a GUI application. This is the default behavior.
  • As an interactive command-line application, which is the default whenever it is used with the --cli option, or if no GUI is available.
  • As a silent configuration mechanism, when all of the appropriate values are provided in the form of command-line arguments.

When the setup utility is run without any arguments, it will start in GUI mode, but will fall back to the interactive command-line mode if no GUI is available. To ensure that it does not attempt to start in GUI mode, use the --cli option. The entire set of options that may be provided are listed below. Note that no arguments will be allowed if the tool is to run in GUI mode, and only the --cli option will generally be given if the interactive command-line setup process is to be used. The remainder of the arguments listed are primarily intended for use in the silent configuration mode.
  • --cli

    This argument indicates that the setup utility should run as a command-line utility rather than a graphical utility. If it is not provided, then no other arguments may be given.

  • -s
    --silentInstall

    This argument indicates that the setup utility should be run in silent mode, in which it will not prompt the user for any information but rather will require that all values be provided through command-line arguments.

  • -p {port}
    --ldapPort {port}

    This argument specifies the port number on which the Directory Server should listen for connections from LDAP clients. The default value for this argument is 389.

  • -S
    --skipPortCheck

    This argument indicates that the setup utility should not make any attempt to determine whether the specified port number is available. If this argument is not provided, then the setup utility will attempt to determine if it is possible to bind to the specified port number for the purpose of establishing client connections, which will verify that the port is not already in use and that the user running the setup tool has the ability to bind to that port.

  • -D {dn}
    --rootUserDN {dn}

    This argument specifies the DN for the initial root user. The default value for this argument is "cn=Directory Manager".

  • -w {password}
    --rootUserPassword {password}

    This argument specifies the password for the initial root user. This option must not be used in conjunction with the --rootUserPasswordFile argument.

  • -W {path}
    --rootUserPasswordFile {path}

    This argument specifies the path to the file containing the password for the initial root user. This option must not be used in conjunction with the --rootUserPassword argument.



  • -b {dn}
    --baseDN {dn}

    This argument specifies the base DN that should be used as the suffix for the database containing user data. The default value for this argument is "dc=example,dc=com". Multiple base DNs may be specified by providing this argument multiple times.

  • -a
    --addBaseEntry

    This argument indicates that the setup utility should add an entry for the base DN into the database. It must not be used in conjunction with either the --importLDIF or the --sampleData options. If none of these options is provided, then the database will be left empty.

  • -i {path}
    --importLDIF {path}

    This argument specifies that the setup utility should populate the database with information from the specified LDIF file. Data may be imported from multiple files by providing this option multiple times, in which case the files will be processed in the order they are provided in the argument list. This argument must not be used in conjunction with either the --addBaseEntry or --sampleData options. If none of these options is provided, then the database will be left empty.

  • -d {numEntries}
    --sampleData {numEntries}

    This argument specifies that the setup utility should populate the database with the specified number of sample user entries. The entries will be generated using the MakeLDIF facility of the import utility, and will be based on the default example.template template. This argument must not be used in conjunction with either the --addBaseEntry or --importLDIF options. If none of these options is provided, then the database will be left empty.

  • -H
    --help

    This argument will cause the setup utility to display usage information and exit without taking any action.
