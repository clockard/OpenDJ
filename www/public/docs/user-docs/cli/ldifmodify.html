The ldifmodify Utility

The ldifmodify utility may be used to make changes to the contents of an LDIF file. It is very similar to the ldapmodify tool, with the exception that it does not perform any LDAP communication but rather operates on a local LDIF file, and also that it does not accept change information on standard input but rather must read all changes from a file.

Also note that in order to make it possible to operate on very large LDIF files with limited amounts of memory, the following limitations will be enforced on the types of changes that can be made:
  • Modify DN operations are not supported. Only add, delete, and modify operations will be allowed.
  • It is not possible to modify or delete an entry that is to be added during the course of processing.

The complete set of arguments that may be used with the ldifmodify tool is listed below. All arguments (with the exception of the --help option) are required.
  • -s {path}
    --sourceLDIF {path}

    This argument specifies the path to the source LDIF file, which contains the data to be updated.

  • -m {path}
    --changesLDIF {path}

    This argument specifies the path to the file containing the changes to apply. The contents of this file must be in LDIF change format.

  • -t {path}
    --targetLDIF {path}

    This argument specifies the path to the target LDIF file, which will consist of the data from the source LDIF with all of the specified changes applied.

  • -H
    --help

    This argument will display usage information for the ldifmodify utility and will exit without attempting to perform any additional processing.
