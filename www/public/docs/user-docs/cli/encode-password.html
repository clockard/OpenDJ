The encode-password Utility

The encode-password utility may be used to interact with the password storage schemes defined in OpenDS. It has three modes of operation:
  • It may be used to list the password storage schemes that are available in the server. In this mode, only the --listSchemes argument is required.
  • It may be used to encode a clear-text password using a given password storage scheme. In this mode, the --storageScheme argument is required, along with a clear-text password either given as an argument with --clearPassword or read from a file via --clearPasswordFile.
  • It may be used to determine whether a given clear-text password is correct for a provided encoded password. In this mode, both a clear-text password (either from --clearPassword or --clearPasswordFile) and an encoded password (either from --encodedPassword or --encodedPasswordFile) are required.

The arguments available for this utility include:
  • -l
    --listSchemes

    This argument may be used to display a list of the password storage schemes that are available for use in the Directory Server. If used by itself, then it will display the names of the password storage schemes that support the user password syntax. If used in conjunction with the --authPasswordSyntax argument, then it will display the names of the password storage schemes that support the authentication password syntax.

  • -c {password}
    --clearPassword {password}

    This argument may be used to provide the clear-text password on which to operate when either encoding a clear-text password or comparing a clear-text password against an encoded password. It must not be used in conjunction with the --clearPasswordFile argument.

  • -F {path}
    --clearPasswordFile {path}

    This argument may be used to specify the path to a file containing the clear-text password on which to operate when either encoding a clear-text password or comparing a clear-text password against an encoded password. It must not be used in conjunction with the --clearPassword argument.

  • -e {password}
    --encodedPassword {password}

    This argument may be used to provide the encoded password to compare against a given clear-text password. If the --authPasswordSyntax argument is also provided, then this password must be encoded using the authentication password syntax; otherwise it should be encoded using the user password syntax. This argument must not be used in conjunction with the --encodedPasswordFile argument.

  • -E {path}
    --encodedPasswordFile {path}

    This argument may be used to provide the encoded password to compare against a given clear-text password. If the --authPasswordSyntax argument is also provided, then this password must be encoded using the authentication password syntax; otherwise it should be encoded using the user password syntax. This argument must not be used in conjunction with the --encodedPassword argument.

  • -s {scheme}
    --storageScheme {scheme}

    This argument specifies the name of the password storage scheme to use when encoding a clear-text password. If the --authPasswordSyntax argument is provided, then this must be the name of a supported authentication password storage scheme; otherwise, it must be the name of a supported user password storage scheme.

  • -a
    --authPasswordSyntax

    This argument indicates that the provided operation should use the authentication password syntax (as defined in RFC 3112), which encodes values in a form like "scheme$authInfo$authValue". If this is not provided, then the user password syntax (which encodes values in a form like "{scheme}value") will be used.

  • -r
    --useCompareResultCode

    This argument indicates that the encode-password tool should use an exit code that indicates whether a given clear-text password matched a provided encoded password. If this argument is given and the clear-text password does match the encoded password, then the tool with have an exit code of 6 (COMPARE_TRUE), or it will have an exit code of 5 (COMPARE_FALSE) if the clear-text password does not match the encoded password, and any other exit code will indicate that the tool failed to complete its processing to make the necessary determination. If this argument is not provided, then an exit code of zero will be used in either case to indicate that the tool completed its processing successfully, or something other than zero if an error occurred.

  • -H
    --help

    This argument will cause the tool to display its usage information and exit immediately without taking any other action.
