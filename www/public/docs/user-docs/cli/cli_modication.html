Command Line Tool Usage Changes

Consistency is critical to the learning and recall of cli's. In the interest of establishing consistency for existing commands and formalizing conventions for use in future commands, below are the modifications that have been made to the existing commands.

Summary

configure-ds
"-j" short option for "jmx port" option is replaced by "-x"
"-W" short option for "root password file" option is replaced by "-j"
"--rootDN" long option for "root User DN" option is replaced by "rootUserDN"

encode-password
"-F" short option for "clear password file" option is replaced by "-f"

export-ldif
"--compressLDIF" long option for "compress" option is replaced by "--compress"
import-ldif
"-S" short option for "randomSeed" option is replaced by "-s" (switch with "skip schema validation" short option)
"-s" short option for "skipSchemaValidation" option is replaced by "-S" (switch with "randomSeed" short option)

ldapcompare
"--host" long option for "hostname" option is replaced by "--hostname"
"--startTLS" long option for "startTLS" option is replaced by "--useStartTLS"
"--noop" long option for "no operation" option is replaced by "--dry-run"

ldapdelete
"--host" long option for "hostname" option is replaced by "--hostname"
"--startTLS" long option for "startTLS" option is replaced by "--useStartTLS"
"--noop" long option for "no operation" option is replaced by "--dry-run"

ldapmodify
"--host" long option for "hostname" option is replaced by "--hostname"
"--startTLS" long option for "startTLS" option is replaced by "--useStartTLS"
"--noop" long option for "no operation" option is replaced by "--dry-run"

ldappasswordmodify
"--hostHost" long option for "hostname" option is replaced by "--hostname"
"--ldapPort" long option for "port" option is replaced by "--port"
"--noop" long option for "no operation" option is replaced by "--dry-run"

ldapsearch
"--host" long option for "hostname" option is replaced by "--hostname"
"--startTLS" long option for "startTLS" option is replaced by "--useStartTLS"
"--noop" long option for "no operation" option is replaced by "--dry-run"

makediff renamed to make-ldif

restore
"--verifyOnly" long option for "no operation" option is replaced by "--dry-run"

start-ds
"-v" short option for "version" option is replaced by "-V"
"-V" short option for "full version" option is replaced by "-F"

statuspanel renamed to status-panel

stop-ds
"--host" long option for "hostname" option is replaced by "--hostname"
"--startTLS" long option for "startTLS" option is replaced by "--useStartTLS"
"--trustStoreFile" long option for "trustStoreFile" option is replaced by "--trustStorePath"

setup
"-i" short option for "ldif file" option is replaced by "-l"
"--importLDIF" long option for "ldif file" option is replaced by "--ldifFile"
"-j" short option for "jmx port" option is replaced by "-x"
"-W" short option for "rootUserPasswordFile" option is replaced by "-j"

windowsservice renamed to windows-service

backup
This utility may be used to back up one or more Directory Server backends.
Usage:  backup {options}
             where {options} include:
-n {backendID}  or   --backendID {backendID}
    Backend ID for the backend to archive
-a  or   --backUpAll
    Back up all backends in the server
-I {backupID}  or   --backupID {backupID}
    Use the provided identifier for the backup
-d {backupDir}  or   --backupDirectory {backupDir}
    Path to the target directory for the backup file(s)
-i  or   --incremental
    Perform an incremental backup rather than a full backup
-B {backupID}  or   --incrementalBaseID {backupID}
    Backup ID of the source archive for an incremental backup
-c  or   --compress
    Compress the backup contents
-y  or   --encrypt
    Encrypt the backup contents
-h  or   --hash
    Generate a hash of the backup contents
-s  or   --signHash
    Sign the hash of the backup contents
-H  or   --help
    Display this usage information		 backup

This utility may be used to define a base configuration for the Directory Server.

             where {options} include:

    Specifies the port on which the Directory Server should
    listen for LDAP communication.

    Specifies the port on which the Directory Server should
    listen for JMX communication.

    Specifies the base DN for user information in the
    Directory Server.

    multiple times.

    Specifies the DN for the initial root user for the
    Directory Server.
-w {rootUserPW}  or   --rootPassword {rootUserPW}
    Specifies the password for the initial root user for the
    Directory Server.
-W {filename}  or   --rootPasswordFile {filename}
    Specifies the path to a file containing the password for
    the initial root user for the Directory Server.
-H  or   --help
    Displays usage information for this program.

             where {options} include:
-b {baseDN}  or   --baseDN {baseDN}
    Specifies the base DN of a backend supporting indexing. Verification is

    complete if each index value references all entries containing that value.

    Specifies that a single index should be verified to ensure it is clean.  An
    index is clean if each index value references only entries containing that
    value.  Only one index at a time may be verified in this way.
