The OpenDS Repository Layout

As noted earlier, the OpenDS source repository has a base URL of https://opends.dev.java.net/svn/opends, but in most cases you will want to check it out using a URL of https://opends.dev.java.net/svn/opends/trunk/opends. The repository contains more than just raw source code, and it will often be necessary to make changes in files other than Java source files. Some of the notable files and directories in the OpenDS source repository include:
  • PRODUCT -- This file contains product and version information that will be used when building OpenDS.

  • build.xml -- This is the primary ant build script that is used to compile and package the server.

  • build.bat -- This is the batch file that should be used to invoke the build process on Windows systems.

  • build.sh -- This is the shell script that should be used to invoke the build process on UNIX-based systems.

  • src/server -- This is the parent directory containing all of the source code for the Directory Server itself.

  • src/dsml -- This is the parent directory containing the source code for the DSML gateway.

  • src/quicksetup -- This is the parent directory containing the source code for the QuickSetup utility.

  • src/statuspanel -- This is the parent directory containing the source code for the Status Panel utility.

  • tests/unit-tests-testng/src -- This is the parent directory for all of the Directory Server unit test source code.

  • tests/unit-tests-testng/resource/config-changes.ldif -- This file contains a set of changes (in LDIF change form) that will be applied to the default server configuration when running the unit tests.

  • tests/integration-tests-testng -- This is the parent directory containing the integration tests for the server.

  • tests/functional-tests -- This is the parent directory containing the functional tests for the server.

  • lib -- This directory contains JAR files that may be needed to build OpenDS and should be included with the Directory Server itself (e.g., the Berkeley DB Java Edition JAR file).

  • ext -- This directory structure contains a number of utilities and libraries that may be needed during the build process but that should not be included with the server itself (e.g., the libraries needed to run ant to build the server, checkstyle to examine the source code, and emma to perform code coverage analysis).

  • resource -- This directory structure contains a number of miscellaneous files that are needed in different ways by different components of the server.

  • resource/bin -- This directory contains a number of shell scripts and batch files that should be provided with the server for use as administrative tools or client utilities.

  • resource/config/config.ldif -- This is the default configuration file that will be used for the server.

  • resource/schema -- This directory contains a number of schema configuration files that will be provided with the server.
