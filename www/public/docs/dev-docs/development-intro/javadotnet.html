The OpenDS Project on java.net

The majority of the OpenDS project is hosted at java.net, which provides a framework for Java-based open source applications that makes it possible for developers to spend more of their time and resources in development and less time in maintaining the project infrastructure. In particular, OpenDS currently takes advantage of the following java.net offerings:
  • The general website provides a place for us to make project information available, including our documentation, FAQ, and other resources
  • The Subversion repository hosts all of the project source
  • The mailing lists provide a mechanism for communicating between developers and users
  • The issue tracker holds information about all of the features, tasks, and defects associated with the project
  • The downloads area provides a mechanism for making files available for download (e.g., our weekly builds)

Joining the OpenDS Project

Much of the project content is available anonymously, but if you wish to participate on the mailing lists, file bugs or feature requests in the issue tracker, or take part in the development of the server, then you will need to join the project. To do this:
  1. If you don't already have one, get an account on java.net. It's free and the only personal information you need to provide is an e-mail address. To create an account, complete the form at https://www.dev.java.net/servlets/Join.

  2. Request a role in the OpenDS project at https://opends.dev.java.net/servlets/ProjectMembershipRequest. You should initially request the "java-enterprise>opends.User" role, which is available for anyone and will give you the ability to participate on OpenDS mailing lists and create issues in the issue tracker.

  3. If you wish to contribute to the project in other ways (e.g., by writing code, documentation, or test cases), then you will need to complete and submit the Sun Contributor Agreement. You may then send an e-mail message to contribution-request@opends.dev.java.net" in order to request the Contributor role.


Important OpenDS Mailing Lists

Once you have joined the project, you will probably also want to subscribe to one or more of our e-mail mailing lists to keep up with what's happening. You can see the full set of mailing lists available at https://opends.dev.java.net/servlets/ProjectMailingListList, where you can also subscribe to any or all of these lists on either a message-by-message basis (in which you'll get each individual e-mail message sent to the list) or in digest mode (in which you'll get one message a day that contains all of that day's traffic on that list).

The mailing list page provides information about all lists in the project, but the most useful lists for someone who wants to participate in the project development include:
  • announce@opends.dev.java.net -- This is an extremely low-traffic list that we will use to provide important announcements related to the OpenDS project.

  • commits@opends.dev.java.net -- This list will automatically be sent an e-mail message whenever someone commits something into the OpenDS Subversion repository. That message will include information about the files that were changed, as well as a diff of those changes.

  • daily-builds@opends.dev.java.net -- This list will automatically be sent an e-mail message whenever the daily build completes, indicating whether the build was successful or failed for some reason.

  • design-discuss@opends.dev.java.net -- This list is intended for use by individuals discussing the design of components in the server before and during their development.

  • dev@opends.dev.java.net -- This list is intended for use by OpenDS developers communicating with each other.

  • issues@opends.dev.java.net -- This list will automatically be sent an e-mail message whenever someone creates or updates an issue in the issue tracker. Note that in most cases, it will not be necessary to join this list because you will automatically be sent a message for updates to issues that you have created, issues assigned to you, or issues in which you are on the interest list.

  • precommit-notification@opends.dev.java.net -- This list is intended for use by individuals for reviewing and commenting on code changes that have been proposed for committing to the repository.

  • technical-questions@opends.dev.java.net -- This list is intended for use by users who wish to ask technical questions of the OpenDS developers.

  • users@opends.dev.java.net -- This list is intended for use by users who wish to ask general questions, provide feedback or comments, or other general messages about OpenDS.

Note that most of the OpenDS mailing lists are configured to automatically allow posts by members subscribed to the list, but to require moderation (explicit approval from a project owner) for any message sent by someone who is not subscribed to that list. This means that if you send a message to a list to which you are not subscribed, then it may take some time for the message to be delivered while it is waiting for moderation, and you may not see any replies because they will generally be delivered only to list members. Therefore, it is strongly recommended that you only send messages to a list if you are subscribed to it.
